# Proposal: Deepen Public Project Posts From Safe Local Insights

## Summary

Expand the short public project posts with non-sensitive technical lessons extracted from updated local project context. The public posts should keep the problem-driven structure but add deeper design tradeoffs, failure modes, evaluation loops, and open-source implementation guidance.

## Scope

- Read local `ai_projects` context as source material for technical abstraction only.
- Update short public posts that were previously summarized too aggressively.
- Keep all sensitive details out of public posts: local paths, private project names, internal scenarios, raw dialogues, ports, credentials, private deployment details, and unreleased ideas.
- Keep local private blog generated content local-only.

## Non-goals

- Do not modify `/mnt/e/workspace/ai_projects`.
- Do not publish raw local notes or conversation excerpts.
- Do not weaken existing public-site privacy checks.
